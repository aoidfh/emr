# 🏥 EMR 병원차트 프로그램

> 실제 병원 업무 흐름을 기반으로 구축된 웹 기반 EMR(전자의무기록) 시스템입니다.  
> 접수실, 진료실, 수납실 등 병원 내 부서별 기능을 UI로 구현하고, MySQL 연동을 통해 환자 정보와 진료 데이터를 처리합니다.
> > **(개발 기간: 2022.7 ~ 2024.07)**  

---

## 📌 프로젝트 개요

- **타입**: 병원 내 EMR 시스템(전자의무기록) 웹 프로그램  
- **역할**: 프론트엔드 개발, 일부 기능의 MySQL 연동  
- **기여 범위**: 전체 UI 설계 및 구현 / 데이터 연동 (초기 단계)

---

## ✅ 상세 구현 내역

### 🧩 공통 기능 (접수실 / 진료실 / 수납실)

- 환자 검색 및 정보 불러오기
- 접수 처리 로직 구현 및 리스트 출력
- 진료 리스트 및 수납 리스트 연동
- 과거 진료 이력 확인 기능
  → 환자 클릭 시 달력에 진료 날짜가 표시되며, 날짜 선택 시
  진료 메모, 처방내역 등의 상세 기록을 리스트 형태로 조회 가능
- 각 기능별로 **MySQL 연동을 통한 기본 CRUD(Create, Read, Update, Delete)** 처리 구현
- 사용자 흐름(UI/UX)을 고려한 화면 간 연동 및 데이터 처리 구성

### 🗂 1. 접수실

- 접수 처리 및 접수 리스트 출력
- 과거 진료 내역 조회

### 🩺 2. 진료실

- 진료 기록 입력 및 저장
- 진료처방 작성 화면 구성

### 💵 3. 수납실

- 진료비 산출 UI 구성 (결제 기능은 제외)

### 🔍 4. 팝업 기능 (약 20여 개)

- 이미지 기반 UI 위주로 구현
- DB 연동 완료 항목:  
  - 환자조회  
  - 접수정보  
  - 처방코드 검색

### ⚙️ 5. 병원관리 / 기초정보 관리

- 일부 항목에 대해 DB 연동 및 기본 CRUD 작업 수행

### 🔐 6. 로그인 / 메인 메뉴

- 커스텀 로그인 UI 구현 (서버 연동 없이 테스트용)
- 메인 메뉴 클릭 시 화면 전환 기능 구현

※ README와 업로드된 파일 모두 주요 기능 중심으로 정리되었으며,  
세부 구현 내용은 보안 및 프로젝트 정책에 따라 일부 제외되었습니다.

## ⚠ 참고 사항

- 현재는 로컬 환경에서 미리보기 용도로 동작하도록 구성되어 있으며, 서버 연동은 포함되어 있지 않습니다.
- 실제 병원 환경 및 보안 이슈로 인해 일부 기능과 파일 경로는 축약 및 가공 후 업로드되었습니다.
- 본 프로젝트는 병원 실사용을 목표로 진행되었으며, 박준영은 **UI 전체 구현 및 초기 데이터 연동 작업을 단독 수행**하였습니다.
- 이후 **회사 사정으로 퇴사하여 후속 개발에는 참여하지 못하였습니다.**
- 전체 프로젝트 구조 및 MySQL 연동 상세 자료는 요청 시 별도로 제공 가능합니다.
